## 0. 元目标:我们到底在设计什么

把"一问一答的执行器",变成一个**有持久状态、会自我改进的闭环系统**:
- 不断**发现问题 → 提方案 → 实验验证 → 得经验教训**,闭环。
- 能**非阻塞地向人提需求**(要信息时记下来继续干别的,不空转等人)。
- 目标:**24 小时持续运行**,无人值守也不跑歪、不空转、不胡说。

一条贯穿始终的核心判断:
> **闭环的循环体本身是最简单的部分。真正决定成败的是两件事:(1) 一个我无法自欺的"外部判定信号"(oracle);(2) 一份能在上下文被压缩后依然存活的持久状态。**
> 没有 (1),系统会"进化"进幻觉里(自己改测试让它通过);没有 (2),跑长了就忘了北极星。

## 1. 业务问题(具体场景)

- 一个复杂业务场景,**2 个页面**,DAU **几千万**。
- 算法可控的**展位 3~4 个**,每个展位做两件事:**排序 + 任务定价**(都已接算法)。
- 任务两类:
  - **活跃任务**:做点击/活跃。
  - **商业化任务**:做收入,分**广告**和**游戏**两类。
- **目标:在对 DAU 影响尽量小的前提下,尽量提升收入。**
- DAU 与收入有换算率,例:**一个任务点击值 0.1 元**。

### 数学化(决定整个系统的重心)

每个曝光机会、每个候选任务 i,给一个**统一货币口径的"混合价值"**:
```
blended_value(i) =
    expected_revenue(i)            若 i 是商业化任务(广告 eCPC×pClick / 游戏期望收入)
    α · pClick(i) · v_click        若 i 是活跃任务(v_click = 0.1 元/点击)
```
- 展位分配 = 按 blended_value 排序填坑;商业化任务再走拍卖定价。
- 全局目标(一天):**maximize Σ revenue,s.t. ΔDAU ≥ −预算**。
- **关键洞察:那个"0.1 元/点击"换算率,本质是这个约束优化的拉格朗日对偶变量 α,是"DAU 的价格"。**
  - α↑ → 活跃任务更易抢坑 → 点击/DAU↑、收入↓(保 DAU)
  - α↓ → 商业化占坑 → 收入↑、DAU 承压(收割收入)
- 所以系统中枢控制量是 α:**自动调 α,让 DAU 卡在允许的预算线上,在这条线上把收入榨到最大。**

## 2. 框架的两种切法(当前共识 = 三层)

### 2a. 最初的七步闭环(一次迭代的内部流程)
① 业务指标数据解读/诊断 → ② 机会识别+可证伪假设 → ③ 优先级 → ④ 干预/实验设计 → ⑤ 灰度执行 → ⑥ 带对照的统计判读 → ⑦ 学习固化 → 回 ①。
- 关键纪律:② 假设必须**先写证伪条件再动手**;④ 改动要小且隔离;⑥ 判定信号必须 agent 伪造不了。
- "数据解读"在闭环里出现**两次**:① 解读世界(哪儿不对)、⑥ 解读实验(改动有没有用)。同一种能力。
- **一个自进化优化系统,本质 80% 是个诊断师。**

### 2b. 用户重构的三层(当前主用模型,比七步更好用)

```
┌─ 第1层:业务目标追踪 + 归因(常驻,只看不动手)──────────────┐
│  北极星指标 + 分解体系,每日/每小时追涨跌 → "哪涨哪跌+为什么"      │
└──────────────────────────┬──────────────────────────────────┘
                           │ 指出:哪里有机会 / 哪里在漏(归因→杠杆=桥)
                           ▼
┌─ 第2层:策略/模型优化闭环(动手)─────────────────────────────┐
│  三条战线,共用"离线找策略 → 上线验证 → 回收数据 → 总结教训":     │
│   1) 工程链路优化   2) 分人群策略   3) 模型优化(autoresearch)    │
│   共同心脏:先用【离线指标】筛出好策略,再上【线上实验】确认       │
└──────────────────────────┬──────────────────────────────────┘
                           │ 经验教训
                           ▼
┌─ 第3层:知识沉淀(团队知识库)────────────────────────────────┐
│  每轮总结进知识库 → 反哺:让第1层归因更准、第2层假设更聪明          │
└─────────────────────────────────────────────────────────────┘
        (第3层回流到第1、2层,三层才闭环)
```

- **第1层追"结果"(我们不直接控),第2层调"杠杆"(我们能控)。归因是连接两层的桥。**
- **离线指标可信度是最大风险**:离线涨了线上不涨(off-policy bias)是头号翻车点。所以第1层的线上结果追踪要**兜底**,做第2层离线判断对不对的最终裁判。
- 第2层三条战线共用闭环,区别只在**杠杆**和**离线尺子**:
  - 1) 工程链路优化:杠杆=链路打分参数/阈值;离线尺子=回放线上打分日志看指标变化。(用户有经验)
  - 2) 分人群策略:杠杆=分人群参数;离线尺子=同样回放但按人群切片。(用户有经验)
  - 3) 模型优化:杠杆=模型本身(特征/结构/重训);离线尺子=离线模型指标(GAUC 等);**autoresearch** 把"提模型改动假设→迭代"自动化,最前沿最难。
- 用户的关键经验(与"oracle 是命门"殊途同归):**必须先有离线指标,让模型搜可调参数,看调完指标怎么变,离线评估筛出好策略,再上线验证。**

## 3. 第1层深入(已讨论到可落地)

第1层目标:**把"业务目标涨跌"自动翻译成"哪个因子/哪个维度导致 + 为什么 + 多确信"。**
三块拼成:**分解树(骨架)→ 自动归因(下钻)→ 双向验证(防归错)。**

### 3a. 分解树(一切归因挂在它上面)
```
【收入树】(乘法分解)
总收入
├─ 商业化收入
│   ├─ 广告收入 = 广告曝光 × 填充率 × CTR × CPC
│   └─ 游戏收入 = 游戏曝光 × 转化率 × 单次转化价值
└─ (活跃侧不直接产收入,但喂 DAU)

【杠杆】(夹在两棵树中间,唯一能动的)
排序:各任务类型在各展位的曝光分配 / 定价:商业化任务出价/eCPM / α:活跃vs商业化权衡

【DAU 树】(加法/留存分解)
DAU = 新增用户 + 回访用户
        └─ 回访 = Σ_cohort(cohort规模 × 留存率)←受活跃任务点击/体验影响
```
每个叶子再按维度切片:**页面(2)× 展位(3~4)× 人群 × 新老用户**。
> 树切得对(因子可定位、且每叶子对应一个杠杆),归因才能落到可行动粒度。**树是地基中的地基。**

### 3b. 自动归因(沿树下钻 + 定位维度 + 判内外因)— 走通的例子「收入 −3%」
1. 顶层下钻:商业化 −3%,活跃侧持平 → 进商业化。
2. 再下钻:游戏收入 −12%,广告持平 → 锁定游戏。
3. 拆因子:游戏收入=曝光×转化×价值;游戏曝光 −10%(填充率掉),其余持平 → 锁定"游戏填充率"。
4. 切维度:下降集中在 展位2 + 新用户。
5. 判内外因:变更日志显示我们没动展位2参数 → 偏外因;查外部=某热门游戏下线,游戏供给少 → 外因确认。
6. 输出:收入 −3%,根因=游戏库存外生减少→展位2新用户填不满→曝光少→收入少;内部杠杆无责。
- **桥**:对应杠杆="展位2 对游戏降门槛 / 广告回填空位" → 这句递给第2层。

### 3c. 双向验证(防伪相关,来自材料闭环 arXiv 2509.18604)
24/7 无人值守最大风险=归因归错(某小时两件事碰巧一起动)。四象限:
```
                  机制能讲通吗?(top-down:领域知识/LLM 说出因果链)
                  能                          不能
数据上显著相关?  ┌──────────────────┬──────────────────────┐
(bottom-up:统计) │ ✅ 确信归因        │ ⚠️ 疑似伪相关→标记/设小实验验 │
   是            │                  │                      │
                 ├──────────────────┼──────────────────────┤
   否            │ 🔍 机制待数据支持  │ ❌ 无关,忽略          │
                 └──────────────────┴──────────────────────┘
```
> 只有"数据相关 + 机制可解"两关都过才算确信归因。这是无人值守还不胡说的根本安全带。

### 3d. 24/7 跑法
- 每小时刷新分解树各节点值,跟基线比(同比上周同日消周内效应 / vs forecast)。
- **噪声地板**(借 CORAL 的 epsilon):每节点有波动带,只有超出才触发下钻归因,**别每小时对抖动喊狼**。
- 触发后分级:外因→知会记录;**内因异常(我们某变更搞坏了)→ 告警,可能回滚**(第1层兜底第2层)。

### 3e. 第1层产出 = 一份"诊断"(交给第2层的接口)
| 字段 | 例子 |
|---|---|
| 指标&偏离 | 总收入 −3%(超噪声地板) |
| 根因因子 | 游戏填充率↓(展位2×新用户) |
| 内外因 | 外因:游戏库存外生减少 |
| 置信度 | 确信(数据+机制双吻合) |
| 对应杠杆(桥) | 展位2对游戏降门槛 / 广告回填空位 |

## 4. 调研结论(论文 + 开源 + CORAL)

调研用 deep-research 跑过(112 agent、对抗式核实)。两个方向:

### 4a. 方向1 论文(已核实)— 范式 → 对应闭环
- **两篇 2025 综述**(arXiv 2507.21046 / 2508.07407):统一 taxonomy("进化什么/何时/如何/在何处")+ "System Inputs–Agent–Environment–Optimisers"反馈环。整体脚手架。
- **EvolveR**(2510.16079):在线交互→**离线自蒸馏出自然语言"策略原则"+结构化三元组**→存带使用/成功计数+剪枝的经验库→RL 更新。**直接命中第3层(经验固化),最可移植**(RL 改权重那段工业不照搬)。
- **材料闭环**(2509.18604):诊断=统计关联/根因研究;**top-down LLM 理论 + bottom-up 数据双向验证防 data fishing**。**直接服务第1层 + 防自欺**(就是 3c 那套)。
- **STOP**(2310.02304,微软,开源):递归自改进,**显式测量"绕过 sandbox / 博弈效用函数"频率**(实际未观察到成功绕过,作残余风险)。**防 reward hacking 的学术先例**。
- **Darwin Gödel Machine**(2505.22954,Sakana+UBC):**agent 归档(archive)开放进化防局部最优** + 经验性 benchmark 验证替代证明。SWE-bench 20%→50%。
- **BioDiscoveryAgent**(ICLR 2025,2405.17631):闭环实验设计;**独立 critic agent 给第二意见**;多工具;不训 ML 模型/不要 acquisition function,靠 LLM 知识选实验。
- **NovelSeek/InternAgent**(上海AI Lab,开源,2505.16938):完整闭环多 agent 自治科研;**非阻塞人类反馈**(自动评分高才介入)。可参考骨架,但缺 ①③⑥⑦。
- **AlphaEvolve / FunSearch / Promptbreeder / Gödel Agent**:LLM+进化式程序搜索,靠**自动评估器(evaluator)**迭代。印证"评估器质量=进化质量上限"。
- 会议:**ICML 2025 有 PRAL(Programmatic Representations for Agent Learning)workshop**(已核实)。其余顶会专门 track 未逐一核实。
- 被驳回的过强说法:"AlphaEvolve 发现可证明正确、跨域超 SOTA 的新算法"(2票驳回,不采纳)。
- **诚实缺口**:全是研究原型,**没有一个自带工业生产要素**(灰度/canary、带对照统计、off-policy/bandit、α 约束优化)。只能借鉴机制,不能搬代码。**真正难的"在线实验+约束优化底座"没人替我们解决。**
- **方向2(开源落地)这轮没做透**(核实预算给了方向1)。已捞到但未深评的相关仓库:**OBP/zr-obp**(off-policy 评估,对应⑥/预筛)、**Ax + BoTorch**(贝叶斯实验设计,对应③/④)、OpenEvolve(程序层)、GEPA/DSPy(prompt/程序层)、Trace/TextGrad(NL反馈优化)。**待补一轮。**

### 4b. CORAL 深读(MIT+NUS+Stanford+Meta Superintelligence Lab;开源 Apache-2.0;arXiv 2604.01658;~724 stars;活跃)
**定位:self-evolving agents 的基础设施,原生集成 Claude Code/Codex/Cursor 等。是调研里唯一"既是研究又能当工程脚手架"的。**

**数据模型(4 类型):**
- `Task`(优化目标)、`Attempt`(commit+agent+score+status+parent_hash 血缘;status∈pending/improved/baseline/regressed/reverted/crashed/timeout)、`Score`(value:float|None + name + explanation + metadata)、`ScoreBundle`(**scores:dict[str,Score] 多指标 + aggregated:float + is_public + feedback:str 自然语言 + compute_aggregated(weights)**)。
- 两个关键点:**ScoreBundle 天生多指标**(能装 revenue/DAU/guardrail + 聚合 + NL 读出);**status="pending" 是一等状态、Score.value 可 None**(原生容忍"提交了还没评出分"=异步实验的落脚点)。

**GRADER(整个系统唯一接口契约):** `async grade(codebase_path, tasks) -> ScoreBundle`。
- daemon:长驻进程 watch `.coral/public/attempts/`;每个 pending 在 **`git worktree add --detach <commit>` 隔离检出**里评(agent 继续提交不扰动);幂等;默认串行;超时强杀→timeout/grader_error。
- 评完跟**该 agent 自己历史最优**比 → improved/baseline/regressed。
- **budget_class:real(计 plateau)/ tune(廉价预筛,不计 plateau)/ grader_error**。← 现成的"贵真测 vs 廉价预筛"二分。

**HEARTBEAT(周期中断驱动的反射引擎):** 注册的 action,各有独立触发器:
- `interval`(每 N eval)→ **reflect**(本地:写实验 note 锚定分数变化+因果+信心+下个预测,蒸馏 skill)、**consolidate**(全局:同主题 3+ note 合成 synthesis/connections/open-questions)、lint_wiki。
- `plateau`(连续 N eval 无改进,**epsilon=噪声地板**,小抖动不重置 streak)→ **pivot**(不是收尾信号,是换方向:攻最难未试想法,区分"被证据否决vs被畏难否决",承诺≥3 eval,声明 lane/posture/budget/abandon-if)。
- **islands**:多子种群各自 attempts/eval_count,岛间迁移=开放进化防局部最优(FunSearch/DGM 同思路)。
- role 文件带 generation 计数 + History 留痕(drift visibility)。

**承重接缝(我们要对接的 5 个):** GraderInterface.grade()、ScoreBundle、budget_class real/tune、status vs own-best + eval_count 时钟、status="pending"。

**根本张力:** CORAL 的 daemon 是"看到 pending → timeout 内**同步**评出 → finalize",默认评测秒~分钟、便宜、可大量重跑。**我们的评测要几天(DAU 几周)。**

## 5. 调研对三层框架的净启发(尤其 24/7)

总判断:**调研不改三层"做什么",但 CORAL 几乎就是"怎么让它 24 小时不停跑"的现成答案。**

- **第1层(归因)** ← 材料闭环的"统计归因 + 理论/数据双向验证防伪相关" + 噪声地板防乱归因。(CORAL 最不覆盖这层——它只 grade 不诊断业务。)
- **第2层(优化)** ← evaluator/off-policy 驱动搜索(战线1/2)+ autoresearch 范式(战线3)+ pivot 防空转 + islands 持续探索 + budget_class real/tune = 离线预筛 vs 线上确认。
- **第3层(沉淀)** ← EvolveR 自然语言经验蒸馏 + CORAL 的 reflect/consolidate/skills/roles 知识库实现。

**24/7 = CORAL 的"操作系统"** — 持续运行需要五样,CORAL 全有:
| 持续运行需要 | CORAL 机制 |
|---|---|
| 不靠人催的时钟/驱动器 | heartbeat |
| 不追噪声乱动 | plateau + epsilon 噪声地板 |
| 卡住会换方向不空转 | pivot |
| 长跑不忘越跑越聪明 | consolidate + role 留痕 |
| 持续开放探索不困局部最优 | islands |
| 要人补信息但不卡住 | (NovelSeek)非阻塞人类反馈 |
> 持续运行 ≠ 一个 while 死跑,= heartbeat 驱动 + 噪声地板防乱动 + pivot 防空转 + islands 持续探索 + 知识库防遗忘 + 非阻塞问人 这一整套"长跑纪律"。

**我们特有、谁都没解决的点 = 慢。** "24 小时持续"对我们 = **三个时钟同时转**:
| 时钟 | 跑什么 | 节拍 | 持续发热? |
|---|---|---|---|
| 小时级 | 第1层追踪+归因 | 每小时 | 一直热 |
| 持续 | 第2层离线:回放/搜参/离线评估/autoresearch | 不停 | **24h 算力主战场** |
| 天~周级 | 第2层线上:A/B 灰度→回收→总结→进知识库 | 实验周期 | 并行等多个实验成熟 |
> 真正能 7×24 满负荷的是**第1层追踪归因 + 第2层离线搜索**(不花真流量)。线上实验是低频最终裁判。**这让"离线指标是命门"更重要:离线层是 24h 主战场,线上是低频裁判。**

## 6. ExperimentGrader 设计(已起草,较深,供后续细化)

把 CORAL 的 grade() 实现成"线上 A/B + α 约束 + 长滞后 DAU holdout"。六个决策:
1. **同步评 → launch+reconcile 两段**:grade() 只注册实验返回 pending(不评分、eval_count 不++);另起 reconciler 守护进程,实验读出成熟后回填 score 才 eval_count++。**eval 时钟与墙钟解耦**:一个 eval=一次成熟读出,plateau 自动变成"连续 N 个实验无显著提升"。
2. **ScoreBundle 装约束**:scores={revenue_delta, dau_delta, guardrail_adload...};aggregated=约束感知混合价值=revenue_delta + α·dau_delta,且任一 guardrail 破线直接判负(防 reward hacking)。
3. **improved 重定义**:从点比较→"约束下显著提升"(blended CI 下界>0 且 guardrail 未破 且 DAU 损失≤预算);inconclusive→grader_error(计 plateau streak 但不动 anchor)。
4. **epsilon=显著性阈值/MDE**;plateau=探索预算耗尽→pivot。
5. **补 CORAL 没有的"稀缺真流量配给层"**:tune=离线预筛(off-policy IPS/DR 或 BO 代理),real=只有预筛 EV 靠前的才升线上 A/B。
6. **DAU 长滞后=双时钟读出**:T+短(数天,revenue+短期DAU代理,可暂定 improved 小步放量);T+长(数周,读永久 holdout,回填同一 attempt 的 dau_delta;若长周期侵蚀超预算→把 improved 改判 **reverted** 并回滚)。α 是对偶控制器,用长周期 DAU 偏差调 α。
- 对账:除"同步 daemon→异步 launch+reconcile"是唯一大改,其余都是在 CORAL 既有契约里填工业语义。难点在 ExperimentGrader 内部:**约束感知评分(α+护栏)、稀缺实验配给、DAU 长滞后双时钟翻案**。
- ⚠️ 用户反馈此节过细、暂时跟不上;后续按需再展开,优先把第1/2层在用户颗粒度上走通。

## 7. 待用户拍板的设计点(非阻塞,先记)
1. **实验容量**:同时能跑多少互不干扰的 A/B?(决定配给层紧张度 + islands 开几个)
2. **α 现在是常量还是已有控制系统在调?**(决定对偶控制器是新建还是接管)
3. **DAU 有没有永久 holdout?**(决定长周期翻案能不能做;做不了只能退用代理指标,更危险)
4. 离线指标与线上结果的历史一致性如何?(off-policy bias 有多大,决定离线层可信度)
